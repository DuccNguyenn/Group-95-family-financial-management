## 1. Tổng Quan Dự Án (Overview)

### 1.1. Dự án này làm gì?

**Family Financial Management** là ứng dụng web giúp các thành viên trong một gia đình cùng quản lý tài chính chung. Mỗi gia đình tạo một **"Space"** (phòng), mời các thành viên vào bằng **mã mời 6 ký tự**. Trong Space, mọi người có thể:

- Ghi nhận **giao dịch thu/chi**
- Đặt **ngân sách** hàng tháng
- Theo dõi **tài khoản** tiền (ví, ngân hàng...)
- Nhận **cảnh báo** khi chi tiêu vượt ngưỡng

#### Code minh họa — Mã hóa mật khẩu

> File: `family-finance-be/src/helper/util.ts`

```typescript
import * as bcrypt from 'bcrypt';
const saltRounds = 10;

export const hashPasswordHelper = async (plainPassword: string) => {
  return await bcrypt.hash(plainPassword, saltRounds);
};

export const comparePasswordHelper = async (plainPassword: string, hashedPassword: string) => {
  return await bcrypt.compare(plainPassword, hashedPassword);
};
```

#### Code minh họa — Tạo Account + User khi đăng ký

> File: `family-finance-be/src/Module/users/users.service.ts`

```typescript
async handleRegister(dto: CreateAuthDto) {
  // 1. Kiểm tra email trùng
  const exists = await this.isEmailExists(dto.email);
  if (exists) throw new BadRequestException(`Email ${dto.email} đã tồn tại`);

  // 2. Hash password
  const passwordHash = await hashPasswordHelper(dto.password);

  // 3. Tạo mã kích hoạt 6 số, hết hạn sau 5 phút
  const code = Math.floor(100000 + Math.random() * 900000).toString();
  const expired = dayjs().add(5, 'minutes').toDate();

  // 4. Tạo Account (thông tin đăng nhập)
  const account = await this.accountModel.create({
    email: dto.email.toLowerCase(),
    passwordHash,
    is_active: false,
    code_verification: code,
    code_expired: expired,
  });

  // 5. Tạo User (thông tin cá nhân) — liên kết với Account
  const user = await this.userModel.create({
    accountId: account._id,
    name: dto.name,
    // spaceId + role = null → chưa vào phòng
  });

  // 6. Gửi email kích hoạt
  await this.mailerService.sendMail({ ... });
}
```

> File: `family-finance-be/src/Module/auths/auths.service.ts`

```typescript
async login(profile: any) {
  const payload = {
    sub: profile._id,           // User._id
    accountId: profile.accountId, // Account._id
    username: profile.email,
    spaceId: profile.spaceId ?? null,
    role: profile.role ?? null,
  };

  return {
    access_token: await this.jwtService.signAsync(payload),
    user: {
      _id: profile._id,
      name: profile.name,
      email: profile.email,
      avatar: profile.avatar,
      spaceId: profile.spaceId ?? null,
      role: profile.role ?? null,
    },
  };
}
```

### 2.3. JWT & Cookie — Chi tiết cơ chế

#### Nội dung bên trong JWT (payload)

Khi BE ký một JWT token, payload chứa:

```json
{
  "sub": "userId (User._id)",
  "accountId": "accountId (Account._id)",
  "username": "email@example.com",
  "spaceId": "spaceId hoặc null",
  "role": "parent | member | null",
  "iat": 1711000000,
  "exp": 1711604800
}
```

#### Cách Server Action set Cookie

> File: `family-finance-fe/lib/action.ts`

```typescript
const setTokenCookie = async (token: string) => {
  (await cookies()).set("token", token, {
    httpOnly: true,       // JS client không đọc được → bảo mật hơn
    secure: process.env.NODE_ENV === "production",
    sameSite: "lax",
    maxAge: 60 * 60 * 24 * 7,  // 7 ngày
    path: "/",
  });
};

// Trong loginAction:
export const loginAction = async (email, password) => {
  const res = await sendRequestServer({ ... });
  if (res?.access_token) {
    await setTokenCookie(res.access_token);  // ← Set cookie tại server
  }
  return res;
};
```

> ⚠️ **Lưu ý:** Cookie được set ở **2 nơi**:
> - **Server Action** (`action.ts`): Set `httpOnly` cookie qua `cookies().set()` — An toàn, JS phía client không đọc được.
> - **Zustand Store** (`auth.store.ts`): Set cookie qua `document.cookie` — Để middleware (Edge Runtime) có thể đọc được token.

#### Cách JwtAuthGuard bảo vệ BE routes

> File: `family-finance-be/src/Module/auths/passport/jwt-auth.guard.ts`

```typescript
@Injectable()
export class JwtAuthGuard extends AuthGuard('jwt') {
  canActivate(context: ExecutionContext) {
    // Kiểm tra route có @Public() không
    const isPublic = this.reflector.getAllAndOverride<boolean>(IS_PUBLIC_KEY, [
      context.getHandler(),
      context.getClass(),
    ]);
    if (isPublic) return true;  // Không cần token
    return super.canActivate(context);  // Chạy JwtStrategy
  }
}
```

> **Guard này được đăng ký GLOBAL** trong `app.module.ts`:
> ```typescript
> providers: [
>   { provide: APP_GUARD, useClass: JwtAuthGuard },
> ]
> ```
> → Mọi route đều bắt buộc có JWT, **trừ** các route đánh dấu `@Public()`.

### 3.2. Zustand Store — Quản lý State

> File: `family-finance-fe/store/auth.store.ts`

```typescript
import { create } from "zustand";

interface AuthState {
  user: IUser | null;
  token: string | null;
  setAuth: (user: IUser, token: string) => void;
  setUser: (user: Partial<IUser>) => void;
  clear: () => void;
}

export const useAuthStore = create<AuthState>()((set, get) => ({
  user: null,
  token: null,

  // Gọi sau khi login/createSpace/joinSpace thành công
  setAuth: (user, token) => {
    set({ user, token });
    // Đồng bộ token ra cookie (để middleware đọc được)
    if (typeof document !== "undefined") {
      document.cookie = `token=${token};path=/;max-age=${60 * 60 * 24 * 7};SameSite=Lax`;
    }
  },

  // Cập nhật user (sau khi đổi profile)
  setUser: (partial) => {
    const current = get().user;
    if (!current) return;
    set({ user: { ...current, ...partial } });
  },

  // Đăng xuất
  clear: () => {
    set({ user: null, token: null });
    if (typeof document !== "undefined") {
      document.cookie = "token=;max-age=0;path=/";
    }
  },
}));
```

#### Giải thích cách đồng bộ

1. **Khi Login thành công:**
   - `loginAction()` (Server Action) set **httpOnly cookie** → an toàn cho middleware.
   - `setAuth()` (Zustand) set **document.cookie** → để `sendRequest()` (client-side) đọc được.
   - Zustand lưu `user` + `token` vào memory → React components re-render.

2. **Khi tải lại trang (F5):**
   - Zustand state bị **reset về null** (không persist).
   - Cookie vẫn còn → middleware vẫn bảo vệ route đúng.
   - `sendRequest()` vẫn đọc được token từ `document.cookie`.

### 3.3. Cách gọi API — Token đính kèm vào Request

#### Client-side — `sendRequest()`

> File: `family-finance-fe/lib/api.ts`

```typescript
// Helper lấy token từ cookie (client-side)
const getTokenFromCookie = (): string | null => {
  if (typeof window === "undefined") return null;
  const match = document.cookie
    .split(";")
    .find((c) => c.trim().startsWith("token="));
  return match ? match.split("=")[1] : null;
};

export const sendRequest = async <T>(props: IRequest): Promise<T> => {
  const token = getTokenFromCookie();  // ← Tự động lấy token từ cookie

  const options: RequestInit = {
    method,
    headers: new Headers({
      ...(token ? { Authorization: `Bearer ${token}` } : {}),  // ← Gắn header
      ...headers,
    }),
    body: JSON.stringify(body),
  };

  return fetch(url, options).then(res => res.json());
};
```

```mermaid
    CheckExp -->|Còn hạn| CheckSpaceProtected{Có spaceId?}
    CheckSpaceProtected -->|Có + vào /space| RedirectDash3["→ /dashboard"]
    CheckSpaceProtected -->|Không + vào route khác| RedirectSpace2["→ /space"]
    CheckSpaceProtected -->|OK| PassThrough["Cho phép truy cập"]
```

#### Code minh họa — Middleware

```typescript
export function middleware(req: NextRequest) {
  const token = req.cookies.get("token")?.value;

  // Protected routes — phải có token
  if (!token) {
    return NextResponse.redirect(new URL("/login", req.url));
  }

  const payload = jwtDecode<any>(token);

  // Token hết hạn → xóa cookie, về login
  if (payload.exp * 1000 < Date.now()) {
    const res = NextResponse.redirect(new URL("/login", req.url));
    res.cookies.delete("token");
    return res;
  }

  // Có spaceId mà vào /space → redirect dashboard
  if (payload.spaceId && isSpaceSetup)
    return NextResponse.redirect(new URL("/dashboard", req.url));

  // Chưa có spaceId mà vào dashboard → redirect /space
  if (!payload.spaceId && !isSpaceSetup)
    return NextResponse.redirect(new URL("/space", req.url));
}
```

#### Code minh họa — Tạo Space (Backend)

> File: `family-finance-be/src/Module/space/space.service.ts`

```typescript
async createSpace(dto: CreateSpaceDto, userId: string, accountId: string, email: string) {
  // 1. Kiểm tra user chưa thuộc phòng nào
  const user = await this.userModel.findById(userId);
  if (user.spaceId) throw new BadRequestException('Bạn đã thuộc 1 phòng rồi');

  // 2. Sinh mã mời không trùng
  const invitedCode = await this.generateUniqueInviteCode();

  // 3. Tạo Space
  const space = await this.spaceModel.create({
    name: dto.name,
    membersId: [new Types.ObjectId(userId)],
    invitedCode,
    alertThresholds: dto.alertThresholds ?? [80, 100],
    createdBy: new Types.ObjectId(userId),
  });

  // 4. Gán user vào Space với role = parent
  await this.userModel.updateOne(
    { _id: userId },
    { spaceId: space._id, role: 'parent' },
  );

  // 5. Ký JWT mới — có spaceId + role
  const access_token = await this.signNewToken({
    sub: userId, accountId, username: email,
    spaceId: space._id.toString(), role: 'parent',
  });

  return { _id: space._id, name: space.name, inviteCode: invitedCode, role: 'parent', access_token, user: {...} };
}
```
